# Локальная база знаний по официальным книгам D&D

Текстовая выжимка из PDF (`officials/*.pdf`), разбитая **постранично** для удобного `grep`.
Генерируется скриптом [`officials/build_kb.py`](../build_kb.py) (PyMuPDF).

## Структура

```
kb/
  phb-2024-en/                 Player's Handbook 2024 (EN)
    toc.md                     оглавление из закладок PDF (с номерами страниц)
    pages/page-001.txt … 387   одна PDF-страница = один файл, номер страницы в имени
  phb-2024-ru/                 Книга игрока 2024 (RU)
    toc.md
    pages/page-001.txt … 388
```

Номер в имени файла = **номер PDF-страницы** (не печатной; печатная обычно на 1 меньше).
В первой строке каждого файла — заголовок с названием книги и номером страницы.

Текст очищен для поиска: удалены BOM/zero-width, неразрывные пробелы приведены к обычным,
переносы слов (мягкий дефис в RU, дефис на конце строки в EN) склеены — поэтому
`grep "обладают"` находит слово, даже если в PDF оно было перенесено.

## Как искать

```bash
# Где упоминается способность (файл = страница):
grep -rl "Скрытная атака" officials/kb/phb-2024-ru/pages/

# С номером строки и контекстом:
grep -rn -B1 -A6 "Огненный шар" officials/kb/phb-2024-ru/pages/

# Английский термин:
grep -rln "Sneak Attack" officials/kb/phb-2024-en/pages/

# Быстро найти раздел и его страницу через оглавление:
grep -n "Ярость" officials/kb/phb-2024-ru/toc.md
```

## Обновление / добавление книг

Положить PDF в `officials/`, дописать строку в `BOOKS` в `build_kb.py`, затем:

```bash
officials/.kbvenv/bin/python officials/build_kb.py
```

PDF-файлы и `.kbvenv/` не версионируются (см. корневой `.gitignore`); текстовая база — версионируется.
