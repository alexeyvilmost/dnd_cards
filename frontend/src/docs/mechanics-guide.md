# Унифицированная механика — руководство

Это руководство описывает **единый язык механик**, на котором в системе выражаются
эффекты, действия, заклинания и черты. Один и тот же JSON-формат (`mechanics`)
интерпретируется одним движком на всём пути персонажа: создание в кузнице → лист →
бой → кампания. Если механику можно описать данными этого формата — её не нужно
программировать: движок применит её сам.

> **Как читать это руководство.** Начните с раздела «Мысленная модель», чтобы понять
> устройство в целом. Дальше идут справочные разделы (поля, `kind`, формулы) — их
> удобно открывать по мере надобности. В конце — рекомендации и разобранные примеры.

---

## 1. Мысленная модель

Каждая сущность (эффект / действие / заклинание / черта) несёт объект `mechanics`.
Он отвечает на три вопроса:

1. **Когда** механика срабатывает — блок `activation` (пассивно, действием,
   реакцией, по триггеру).
2. **Что** происходит — массив `effects` (в конструкторе — `interactions`): список
   исходов. Каждый исход имеет способ **разрешения** (`resolution`: бросок атаки,
   спасбросок, проверка или «автоматически») и набор **пейлоадов** (`payload`) —
   атомарных изменений состояния (урон, состояние, модификатор, выдача владения…).
3. **Сколько** и **как долго** — стоимость (`cost`), ограничения использований
   (`uses`), длительность (`duration`).

Разрешение бывает двух видов, и это важно понимать:

- **Билд-разрешение** (сборка персонажа). Пассивные гранты — владения, заклинания,
  черты, характеристики, числовые модификаторы листа — применяются при сборке
  персонажа и формируют его лист. Отвечает `resolveCharacterRules`.
- **Рантайм-разрешение** (во время игры). Активные действия, реакции и триггеры
  исполняются на листе/в бою: бросаются кости, тратятся ресурсы, наносится урон,
  накладываются состояния. Отвечает `executeAction`.

Один и тот же `payload` (например `modifier`) может участвовать в обоих путях —
движок применяет его единообразно.

## 2. Структура верхнего уровня

Объект `mechanics` (обязательные поля — `id`, `name`, `kind`, `activation`):

| Поле | Тип | Назначение |
|---|---|---|
| `id` | строка `^[a-z0-9][a-z0-9-]*$` | Технический идентификатор механики. |
| `name` | строка | Английское/служебное имя. |
| `russian_name` | строка | Отображаемое русское имя (опц.). |
| `kind` | `passive_effect` \| `action` \| `spell` \| `trait` | Тип носителя. |
| `source` | строка | Происхождение: `class:<id>:<lvl>`, `species:<id>`, `feat:<id>`, `item:<id>`, `spell`, … (опц.). |
| `description` | строка | Человекочитаемое описание. |
| `activation` | объект | **Когда** срабатывает. См. §3. |
| `interactions` / `effects` | массив | **Что** происходит. См. §4. |
| `targeting` | объект | Цель/форма/дальность (описательно). См. §9. |
| `duration` | объект | Длительность наложенного эффекта. См. §8. |
| `uses` | объект | Лимит использований. См. §7. |
| `requirements` | массив | Требования к носителю (проверяются частично). |
| `ammo` | строка/объект | Боеприпас для дальнобойного оружия. |

> **Термин.** В сохранённом контенте массив исходов называется `effects`, в блочном
> конструкторе — `interactions`. Это одно и то же поле; движок понимает оба имени.

## 3. `activation` — когда срабатывает

| Поле | Значения | Смысл |
|---|---|---|
| `mode` | `passive` \| `active` \| `reaction` \| `triggered` | Режим срабатывания (обязательно). |
| `cost` | массив `cost` | Что тратится при активации (см. §7). |
| `trigger` | объект | Для `triggered`/`reaction`: событие-триггер (см. §3.1). |
| `while` | `equipped` \| `carried` \| `attuned` | Для предметов: пока надет / в сумке / настроен. Нет → `equipped`. |
| `consumes_self` | boolean | Действие расходует сам предмет (зелье). |
| `casting_time`, `range`, `components` | — | Справочные поля заклинаний (v/s/m). |

- **`passive`** — применяется всегда, пока активна (владения, модификаторы,
  сопротивления). Идёт через билд-разрешение.
- **`active`** — игрок нажимает на действие/заклинание на листе; тратится `cost`.
- **`reaction`** — предлагается игроку в ответ на событие (если задан `cost`).
- **`triggered`** — исполняется автоматически по событию (без стоимости).

### 3.1. `trigger` — событие

| Поле | Значения |
|---|---|
| `event` | `hit`, `miss`, `crit`, `damage_taken`, `spell_cast`, `reduced_to_0_hp`, `turn_start`, `turn_end`, `short_rest`, `long_rest` (эмитятся движком) + запланированные: `attack_roll_made`, `damage_dealt`, `saving_throw_made`, `forced_save`, `ability_check_made`, `creature_enters/leaves_reach`, `creature_moves`, `condition_applied`, `initiative_roll`, `on_acquire`, `level_gained` |
| `timing` | `before` \| `during` \| `after` \| `replaces` |
| `subject` | `self` \| `ally` \| `enemy` \| `attacker` \| `target` \| `any_creature` |
| `circumstances` | массив предикатов (см. §6) |

> **Статус.** Реально эмитятся (то есть могут запустить `triggered`/`reaction`):
> `hit`, `miss`, `crit`, `damage_taken`, `spell_cast`, `reduced_to_0_hp`,
> `turn_start`, `turn_end`, `short_rest`, `long_rest`. Остальные события зарезервированы
> схемой и заработают вместе с боевой моделью (позиции/несколько существ). Пока их не
> используйте как рабочий триггер.

## 4. `effects` / `interactions` — что происходит

Массив исходов. Каждый элемент — либо **interaction** (разрешение + пейлоады), либо
**choice верхнего уровня** (выбор игрока, разворачивающийся в пейлоады).

Пассивная механика почти всегда выглядит так:

```json
{
  "activation": { "mode": "passive" },
  "effects": [
    { "resolution": "auto", "result": [ { "kind": "grant_proficiency", "prof": "skill", "value": "perception" } ] }
  ]
}
```

`resolution: "auto"` означает «без броска» — пейлоады из `result` применяются напрямую.

---

## 5. `resolution` — способ разрешения

Каждый interaction задаёт, **как** проверяется исход, и по результату применяет один
из наборов пейлоадов (`outcome` — массив `payload`).

### 5.1. `auto` — без броска
Пейлоады применяются напрямую. Ветка результата — `result`.
Это основной способ для пассивок, самолечения, выдачи ресурсов, наложения состояния
без спасброска.

```json
{ "resolution": "auto", "result": [ { "kind": "healing", "amount": "2d4 + spellcasting" } ] }
```

### 5.2. `attack_roll` — бросок атаки
Бросок к20 + модификатор против КЗ цели.

| Поле | Смысл |
|---|---|
| `ability` | Характеристика атаки: `auto` (по оружию/заклинанию), `spellcasting`, или явная. |
| `attack_kind` | `weapon_melee` \| `weapon_ranged` \| `unarmed` \| `spell_melee` \| `spell_ranged`. |
| `advantage` | `none` \| `advantage` \| `disadvantage`. |
| `on_hit` / `on_crit` / `on_miss` | Наборы пейлоадов по результату. |

```json
{
  "resolution": "attack_roll", "ability": "auto",
  "on_hit":  [ { "kind": "damage", "dice": "weapon" } ],
  "on_crit": [ { "kind": "damage", "dice": "weapon" } ]
}
```

### 5.3. `save` — спасбросок
Цель бросает спасбросок против СЛ; по результату — `on_fail` / `on_success`.

| Поле | Смысл |
|---|---|
| `dc` | Сложность (формула, напр. `8 + prof_bonus + spellcasting`). |
| `who` | Кто бросает: `target` (обычно) \| `self`. |
| `on_fail` / `on_success` | Наборы пейлоадов. |

Урон «половина при успехе» задаётся на самом пейлоаде урона: `"on_success": "half"`.

```json
{
  "resolution": "save", "dc": "8 + prof_bonus + spellcasting", "who": "target",
  "on_fail":    [ { "kind": "damage", "dice": "8d6", "type": "fire", "on_success": "half" } ],
  "on_success": [ { "kind": "narrative", "description": "Половина урона" } ]
}
```

### 5.4. `ability_check` — проверка характеристики
Для активных проверок (Толчок, Подножка, снятие захвата и т.п.).

| Поле | Смысл |
|---|---|
| `skill` | Навык(и) проверки (строка или массив). |
| `mode` | `dc` (против сложности) \| `contest` (состязание). |
| `dc` | Сложность (для `mode: dc`). |
| `contest_vs` | Против каких навыков цели состязание (для `mode: contest`). |
| `on_success` / `on_fail` / `result` | Наборы пейлоадов. |

## 6. `payload` — атомарные изменения (`kind`)

Пейлоад — единица изменения состояния. Ниже — все `kind` со статусом поддержки:

- **Полностью** — движок исполняет; можно писать тесты на реальных данных.
- **Частично** — часть работает, часть — пометка/нарратив (см. примечание).
- **Заглушка** — валидно по схеме, но пока не исполняется (нужна доработка движка).

### 6.1. Урон и лечение

| kind | Поля | Статус |
|---|---|---|
| `damage` | `dice` (формула/`weapon`), `type` (тип урона), `scaling`, `on_success` (`half`/`none`), `per_dart` | Полностью |
| `healing` | `amount` (формула), `scaling` | Полностью |
| `temp_hp` | `amount` (формула) — берётся максимум, не сумма | Полностью |

`dice: "weapon"` разворачивается в кость оружия в руке (+ мод характеристики +
зачарование). Дополнительный урон — **отдельным** пейлоадом (не `weapon + 2d6`):

```json
"on_hit": [
  { "kind": "damage", "dice": "weapon" },
  { "kind": "damage", "dice": "2d6", "type": "necrotic" }
]
```

Скейлинг апкаста/заговора:
```json
{ "kind": "damage", "dice": "1d10", "type": "fire", "scaling": { "per": "spell_slot_above", "dice": "1d10" } }
```
`per`: `spell_slot_above` (апкаст: +кость за круг выше базового) или `character_level`
(рост заговора на 5/11/17 уровне).

### 6.2. Состояния и сопротивления

| kind | Поля | Статус |
|---|---|---|
| `condition` | `value` (id состояния), `op` (`apply`/`remove`), `duration`, `save_ends` | Полностью |
| `resistance` | `damage_type`, `value` (`resistance`/`immunity`/`vulnerability`) | Полностью |

```json
{ "kind": "condition", "value": "frightened", "op": "apply", "save_ends": { "ability": "wis" } }
{ "kind": "resistance", "damage_type": "bludgeoning", "value": "resistance" }
```

### 6.3. Модификаторы значений

`modifier` — самый универсальный пейлоад: меняет бросок или производное значение.

| Поле | Значения |
|---|---|
| `applies_to.roll` | `attack`, `damage`, `ability_check`, `saving_throw`, `ac`, `speed`, `spell_dc`, `initiative`, `carry`, `max_hp` |
| `applies_to.filter` | Уточнение (напр. `{ "ability": "dex" }`). |
| `op` | `add`, `set`, `advantage`, `disadvantage`, `multiply`, `upgrade`, `downgrade` |
| `value` | Формула. |
| `priority` | Порядок свёртки (для `set`/`multiply`/`upgrade`/`downgrade`). |
| `when` | Условия (см. §6.7). |
| `duration` | Для временных баффов. |

```json
{ "kind": "modifier", "applies_to": { "roll": "ac" }, "op": "add", "value": "1" }
{ "kind": "modifier", "applies_to": { "roll": "max_hp" }, "op": "add", "value": "2 * self_level" }
{ "kind": "modifier", "applies_to": { "roll": "ac" }, "op": "upgrade", "value": "16" }
```

Статус: **полностью**. Алгебра `set`/`multiply`/`upgrade`/`downgrade` со свёрткой по
`priority` работает для КЗ; для скорости/спасбросков/навыков применяются аддитивные
модификаторы. `op: reroll` схемой предусмотрен, но в сборе модификаторов пока не
используется.

### 6.4. Значения-методы и прямая установка

| kind | Поля | Статус |
|---|---|---|
| `value_method` | `target` (характеристика/значение), `formula` | Полностью для характеристик (берётся максимум из методов; потолок 20 не режет). |
| `set_value` | `target` (`hp`/`temp_hp`/`max_hp`/`ac_base`/ресурс), `formula` | Полностью для hp/temp_hp/max_hp/известного ресурса; `ac_base` — метод КЗ. |

`value_method` — модель «набор методов расчёта, берём максимум подходящего»
(напр. Пояс силы огра: СИЛ = max(база, 19)):
```json
{ "kind": "value_method", "target": "str", "formula": "19" }
```

### 6.5. Ресурсы, предметы, движение

| kind | Поля | Статус |
|---|---|---|
| `resource` | `op` (`grant`/`restore`/`spend`), `id`, `amount`, `level` | Полностью |
| `add_item` | `card_id`, `qty`, `name` | Полностью |
| `movement` | `value` (`push`/`pull`/`teleport`/`extra_speed`/`double`/`knock_prone`), `distance` | Частично — логируется, но без модели позиций эффекта нет. |

### 6.6. Билд-гранты (применяются при сборке персонажа)

| kind | Поля | Статус |
|---|---|---|
| `grant_proficiency` | `prof` (`skill`/`tool`/`saving_throw`/`weapon`/`armor`/`language`), `value` | Полностью |
| `grant_expertise` | `value` | Полностью |
| `grant_language` | `value` | Полностью |
| `grant_feat` | `value` (slug черты) | Полностью |
| `grant_spell` | `value` (slug/uuid), `level_gate`, `ability`, `freeuse` | Полностью |
| `grant_ability_score` | `ability`, `amount` | Полностью (потолок 20) |
| `grant_sense` | `sense` (`darkvision`/`tremorsense`/`blindsight`/`truesight`), `range` | Полностью |
| `grant_speed` | `mode` (`walk`/`fly`/`swim`/`climb`), `value` | Полностью |
| `grant_effect` | `value`/`values` (slug'и эффектов) | Полностью (разворачивается как под-эффект) |
| `grant_action` | `value`/`values` (slug'и действий), `level_gate` | Частично — даёт доступ к действию на листе, но пул использований пока не гейтится. |

`level_gate` (или `min_level`) на гранте откладывает выдачу до нужного уровня —
так способности видов/классов распределяются по уровням.

**`grant_spell.freeuse`** — заклинание можно кастовать без ячейки из ограниченного
пула (по умолчанию 1×/долгий отдых). Формы: `true`, число, или объект
`{ count, recharge, level }`. Работает и нативно, и внутри `choice.grant`:
```json
{ "kind": "grant_spell", "value": "misty_step", "level_gate": 5, "freeuse": { "count": 1, "recharge": "long_rest" } }
```

### 6.7. Прочее

| kind | Статус | Примечание |
|---|---|---|
| `narrative` | Полностью | Текст-описание того, что движок не исполняет. Используйте для честной фиксации ненадёжной части. |
| `choice` | Полностью | Выбор игрока (см. §10). |
| `boon` | Частично | Чип-талон + нарратив; кость вводится диалогом кубов вручную. |
| `reroll` | Частично | Только нарратив-инструкция (переброс — в диалоге кубов). |
| `transform` | Частично | Чип облика; стат-блок зверя не подменяется. |
| `variable` | Заглушка | Рантайм-мутация переменных пока не реализована. |
| `set_die` | Заглушка | Не реализован. |

## 7. Стоимость и ресурсы

Стоимость активации — массив `cost` в `activation`. Каждая запись:

| Поле | Смысл |
|---|---|
| `resource` | Ключ ресурса (строка). |
| `amount` | Сколько тратится (формула, по умолчанию 1). |
| `level` | Для `spell_slot` — круг ячейки (`0` = заговор). |
| `card_id` | Для `resource: "item"` — id расходуемой карты. |

`resource` — **открытый** список. Частые ключи: `action`, `bonus_action`,
`reaction`, `free_action`, `movement`, `spell_slot`, `rage`, `focus`,
`superiority_die`, `sorcery_points`, `channel_divinity`, `wild_shape`,
`lay_on_hands`, `luck_points`, `hp`, `hit_die`, `item`. Допустим любой ключ из
справочника ресурсов.

- **Ячейки заклинаний.** `{ "resource": "spell_slot", "level": 2 }` тратит ячейку
  2-го круга. Каст доступен при наличии слота этого или **старшего** круга (апкаст).
- **Предмет как стоимость.** `{ "resource": "item", "card_id": "<id>" }` тратит карту
  из инвентаря (боеприпас, зелье).
- **Пул использований действия.** Поле `uses` создаёт виртуальный ресурс, привязанный
  к самому действию (см. §7.1) — тратится и восстанавливается как обычный ресурс.

### 7.1. `uses` — лимит использований

```json
"uses": { "count": "1 + prof_bonus", "per": "long_rest" }
```

| Поле | Значения |
|---|---|
| `count` | Максимум (формула). |
| `per` | `turn` \| `round` \| `short_rest` \| `long_rest` \| `day` |

Долгий отдых восстанавливает все пулы; короткий — только помеченные `short_rest`.

## 8. `duration` — длительность

Для наложенных состояний и временных модификаторов.

| Поле | Значения |
|---|---|
| `type` | `instantaneous`, `rounds`, `minutes`, `hours`, `while_active`, `until_long_rest`, `until_dispelled`, `permanent`, `until_start_of_next_turn`, `until_end_of_turn` |
| `amount` | Число (для `rounds`/`minutes`/`hours`). |
| `concentration` | boolean — требует концентрации. |

> **Статус.** Тикают по ходам: `rounds`, `until_start_of_next_turn`,
> `until_end_of_turn`. `minutes`/`hours`/`while_active`/`until_dispelled`/`permanent`
> живут до ручного снятия/отдыха (таймеров реального времени нет).

Концентрация: каст `concentration`-заклинания вешает чип и вытесняет прежнюю
концентрацию; урон запускает проверку ТЕЛ (СЛ = max(10, урон/2)).

---

## 9. `targeting` — цель и форма

Описательный блок (используется превью/интерфейсом; на разрешение пока не влияет —
площадные эффекты появятся с боевой моделью).

| Поле | Значения |
|---|---|
| `shape` | `self`, `single`, `multi`, `area`, `aura` |
| `range` | Дальность (строка). |
| `max_targets` | Число или формула. |
| `area` | `{ kind: sphere/cube/cone/line/cylinder/emanation, size }` |
| `filter` | `any`, `enemy`, `ally`, `ally_and_self`, `creature_type:undead`, … |

---

## 10. `choice` — выбор игрока

Выбор разворачивается в пейлоады через **шаблон-грант** (`grant`) или через
пер-вариантные гранты (`options.items[].grants`). Точка применения общая с нативными
грантами, поэтому `freeuse`, `level_gate` и т.п. работают и внутри выбора.

| Поле | Смысл |
|---|---|
| `id` | Ключ выбора (в `resolved_choices` персонажа). |
| `prompt` | Текст запроса. |
| `count` | Сколько выбрать (по умолчанию 1). |
| `options.source` | `skill`, `tool`, `ability`, `saving_throw`, `language`, `feat`, `spell`, `damage_type`, `instrument`, `artisan_tool`, `subfeature`, `explicit` |
| `options.filter` | Уточнение пула (напр. `{ classes:["волшебник"], levels:[0] }`). |
| `options.categories` | Для `source: feat` — категории черт. |
| `options.items` | Для `explicit`/`subfeature` — явный список вариантов с `grants`. |
| `grant` | Шаблон гранта: выбранное значение подставляется в `value`. |
| `context` | `build` \| `level_up` (в кузне) \| `in_play` (на листе во время игры). |

```json
{
  "kind": "choice", "id": "magic_initiate_l1", "count": 1,
  "prompt": "Выберите заклинание 1 круга",
  "options": { "source": "spell", "filter": { "classes": ["волшебник"], "levels": [1] } },
  "grant": { "kind": "grant_spell", "label": "spellbook", "freeuse": { "count": 1, "recharge": "long_rest" } }
}
```

## 11. `when` — предикаты-условия

Массив `when` на модификаторе (и `circumstances` на триггере) включает пейлоад только
при выполнении условий. **По умолчанию — «закрыто»**: нераспознанный предикат даёт
`false` (пейлоад не применяется), кроме `narrative` (всегда `true`).

**Реализованы:** `any_of`, `all_of`, `not`, `item_equipped`, `item_carried`,
`attuned`, `you_have_condition`, `target_has_condition`, `has_advantage`,
`d20_equals`, `narrative`.

**Пока инертны** (валидны схемой, движок → `false`): `attack_is`, `ally_within`,
`creature_within`, `target_type`, `target_wears`, `wielding`, `unseen_by_target`,
`proficiency_skill_in`. Им нужна боевая модель/позиции.

```json
{
  "kind": "modifier", "applies_to": { "roll": "ac" }, "op": "add", "value": "1",
  "when": [ { "kind": "item_equipped", "value": "shield" } ]
}
```

> **Важно.** `when`-предикаты гейтят на боевом пути (`executeAction`), но на листе
> (расчёт КЗ/HP) `evalCtx` не всегда передаётся — там условие не блокирует. Не
> опирайтесь на `when` для чисто-листовых значений, пока это не закрыто.

## 12. Формулы и кости

Все поля, помеченные как **формула**, принимают число или строку-выражение. Строку
вычисляет единый парсер (`engine/formula.ts`).

### 12.1. Синтаксис

| Конструкция | Пример | Значение |
|---|---|---|
| Число | `3` | Константа. |
| Кость | `2d6` | Бросить 2 кости д6 (в логе — «к6»). |
| Операторы | `+ - * / ( )` | Арифметика со скобками. |
| Функции | `min(a, b)`, `max(a, b)` | Минимум/максимум. |

### 12.2. Токены (значения персонажа)

| Токен | Значение |
|---|---|
| `str`, `dex`, `con`, `int`, `wis`, `cha` | Модификатор характеристики. |
| `prof_bonus` (или `prof`) | Бонус мастерства. |
| `self_level` | Уровень персонажа. |
| `class_level:<id>` | Уровень конкретного класса. |
| `spellcasting` | Модификатор заклинаний. |
| `spell_slot_above` | На сколько кругов выше базового скастовано (апкаст). |
| `rage_bonus`, `character_speed` | Бонус ярости, скорость персонажа. |
| `<имя переменной>` | Переменная персонажа: число → плоский модификатор, кость(и) → бросок (напр. `rage_damage_modifier`, `martial_arts_die`). |

Скейлинг-сокращение: `class_level:<id>/<делитель> d<грани>` — бросить
`ceil(уровень/делитель)` костей (напр. `class_level:cleric/2 d8`).

### 12.3. Маркеры

`weapon` и `auto` — не числа, а **маркеры**: `weapon` разворачивается в кость оружия
в руке, `auto` — «характеристика по умолчанию». Их **нельзя** складывать/умножать с
числами (`weapon + 2` — ошибка; используйте отдельный пейлоад урона).

### 12.4. Где поддерживаются формулы

Поля, принимающие формулу: `damage.dice` и `scaling.dice`, `healing.amount`,
`temp_hp.amount`, `modifier.value`, `movement.distance`, `resource.amount`,
`set_value.formula`, `value_method.formula`/`value`, `grant_speed.value`,
`interaction.dc`, `transform.cr_max`, `cost.amount`, `uses.count`, `freeuse.count`.

### 12.5. Мягкая деградация

Если формула не вычисляется (опечатка, неактивная переменная, неизвестный токен) —
движок **не падает**: он пропускает этот пейлоад/эффект с записью в лог. Реальный
персонаж не ломается из-за одной битой формулы. Это удобно, но означает, что молчаливо
пропущенный эффект легко не заметить — проверяйте числа тестами.

## 13. Как происходит разрешение (конвейер)

### 13.1. Билд (сборка персонажа)
`resolveCharacterRules` обходит механику всех эффектов/действий/черт носителя
(с учётом `level_gate`), разворачивает `choice` по сделанным выборам и применяет
билд-гранты и числовые `modifier`/`value_method` → формирует `CharacterRuleState`
(владения, заклинания, характеристики, КЗ, HP, скорость, ресурсы). Это состояние —
основа листа.

### 13.2. Рантайм (исполнение действия)
`executeAction(state, mechanics, ctx)`:
1. Читает `activation.cost`, проверяет и списывает ресурсы (`canPay`/`pay`).
2. Для каждого interaction выполняет `resolution` (бросок/спас/проверка/авто) и
   применяет соответствующую ветку `outcome` через единый роутер пейлоадов.
3. Эмитит события (`hit`, `spell_cast`, `damage_taken`…), которые могут запустить
   `triggered`/`reaction`-механики (шина событий с бюджетом каскада).
4. Возвращает новое состояние + журнал событий. Урон по цели проходит конвейер
   сопротивление → концентрация → реакции.

### 13.3. Двусторонний контекст
Пейлоад с `who: "target"` меняет состояние цели, а не исполнителя; спасбросок берёт
модификаторы **цели**. Это позволяет одному формату описывать и «я лечу себя», и
«накладываю состояние на врага».

## 14. Рекомендации (best practices)

1. **Моделируй данными, не кодом.** Если поведение выразимо этим форматом — не пиши
   отдельный код под экран. Один интерпретатор работает на всех этапах.
2. **Дроби эффекты гранулярно.** Не сливай разное в один пейлоад: игрок должен видеть
   источник каждого бонуса. Изменение переменной — отдельным пейлоадом.
3. **Честно помечай нарратив.** Ненадёжную/невыразимую часть выноси в `narrative`, а
   не имитируй. Лучше «без механики», чем с неправильной.
4. **Дополнительный урон — отдельный пейлоад** с явным `type`, а не `weapon + Nd6`.
5. **Проверяй числа, а не только валидность схемы.** Из-за мягкой деградации формул
   молчаливо пропущенный эффект проходит линт; тесты на конкретные значения (HP, КЗ,
   переменные) ловят это.
6. **Сначала — самый узкий `resolution`.** Пассив → `auto`; спас/атака — только когда
   действительно есть бросок.
7. **`level_gate` — для уровневой прогрессии**, а не отдельные копии эффекта.
8. **Не опирайся на инертные предикаты/заглушки** (§6.7, §11). Проверяй статус.

## 15. Полные примеры

### 15.1. Пассивный модификатор (Боевой стиль «Оборона»: +1 КЗ в доспехе)
```json
{
  "activation": { "mode": "passive" },
  "effects": [ { "resolution": "auto", "result": [
    { "kind": "modifier", "applies_to": { "roll": "ac" }, "op": "add", "value": "+1", "source": "Оборона" },
    { "kind": "narrative", "description": "Бонус действует, только пока вы носите доспех." }
  ] } ]
}
```

### 15.2. Заклинание-атака с уроном (Огненный снаряд)
```json
{
  "activation": { "mode": "active", "cost": [ { "resource": "action" } ] },
  "effects": [ {
    "resolution": "attack_roll", "attack_kind": "spell_ranged", "ability": "spellcasting",
    "on_hit": [ { "kind": "damage", "dice": "1d10", "type": "fire",
                  "scaling": { "per": "character_level", "dice": "1d10" } } ]
  } ]
}
```

### 15.3. Заклинание со спасброском и половиной урона
```json
{
  "activation": { "mode": "active", "cost": [ { "resource": "action" }, { "resource": "spell_slot", "level": 3 } ] },
  "effects": [ {
    "resolution": "save", "dc": "8 + prof_bonus + spellcasting", "who": "target",
    "on_fail":    [ { "kind": "damage", "dice": "8d6", "type": "fire", "on_success": "half" } ]
  } ]
}
```

### 15.4. Выданное заклинание с бесплатным использованием
```json
{
  "activation": { "mode": "passive" },
  "effects": [ { "resolution": "auto", "result": [
    { "kind": "grant_spell", "value": "detect_magic", "level_gate": 3,
      "freeuse": { "count": 1, "recharge": "long_rest" } }
  ] } ]
}
```

### 15.5. Выбор навыка при создании
```json
{
  "activation": { "mode": "passive" },
  "effects": [ {
    "kind": "choice", "id": "primal_skill", "count": 1, "prompt": "Выберите навык",
    "options": { "source": "explicit", "items": [ { "id": "nature", "name": "Природа" }, { "id": "insight", "name": "Проницательность" } ] },
    "grant": { "kind": "grant_proficiency", "prof": "skill" }
  } ]
}
```
