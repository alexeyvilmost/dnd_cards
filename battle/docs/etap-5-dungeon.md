# Этап 5 — Dungeon Crawl MVP

Статус: завершён.

## Что сделано

### Бэкенд
- Добавлен слой хранения забегов:
  - `backend/runs_repo.py` (`battle_runs`).
- Добавлена оркестрация Dungeon Crawl:
  - `backend/dungeon.py`:
    - старт забега,
    - генерация комнаты с нарастающей сложностью,
    - запуск боя,
    - резолв комнаты (победа/поражение),
    - награды (gold + XP учёт),
    - магазин между комнатами.
- Добавлены API забегов:
  - `GET /runs`
  - `POST /runs/start`
  - `GET /runs/{id}`
  - `POST /runs/{id}/next-room`
  - `POST /runs/{id}/resolve`
  - `GET /runs/{id}/shop`
  - `POST /runs/{id}/shop/buy`
- В магазин встроена интеграция с предметами:
  - если доступен `dnd_cards` API — офферы собираются из реальных карт;
  - при покупке карточки предмет сразу импортируется в экипировку листа персонажа.

### Фронтенд
- Реализована страница `DungeonPage`:
  - выбор персонажа и старт забега,
  - запуск следующей комнаты,
  - встроенный `CombatView` (с автоходами монстров),
  - резолв комнаты после боя,
  - отображение статуса забега, золота, XP за забег,
  - магазин между комнатами и покупка предметов.
- В `api/client.ts` добавлен `runsApi`.

### Приёмка
- `scripts/test_battle_stack.py` дополнен блоком **Dungeon Crawl MVP**:
  - старт забега,
  - генерация комнаты,
  - автопрохождение боя,
  - резолв комнаты,
  - проверка перехода состояния забега.

## Как проверить

```bash
cd battle
python3 scripts/test_battle_stack.py --no-frontend
```

Ожидаемо:
- блок `Dungeon Crawl MVP` — все `PASS`,
- итог `All checks passed`.

### Вручную в UI
1. Откройте `/dungeon`.
2. Выберите персонажа и нажмите «Начать забег».
3. Нажмите «Войти в следующую комнату».
4. Проведите бой (монстры ходят автоматически).
5. После завершения нажмите/дождитесь резолва комнаты.
6. Проверьте:
   - увеличились `gold` и XP прогресс,
   - статус забега перешёл в `between_rooms` (или `completed` на финальной глубине),
   - между комнатами доступен магазин.

## Примечания
- Магазин работает в двух режимах:
  - с реальными офферами из `dnd_cards` (если API поднят),
  - пустой/ограниченный режим при недоступном items API.
- MVP-ограничения: фиксированная глубина, простая генерация встреч, базовая экономика.
