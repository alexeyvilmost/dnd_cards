# Этап 1 — Лист персонажа и прогрессия

Статус: завершён.

## Что сделано

### Бэкенд
- `backend/progression.py` — модуль прогрессии (D&D 2024, MVP):
  - XP-пороги (1→0, 2→300, 3→900), кап уровня 3.
  - Point-buy (27 очков, 8–15) с валидацией.
  - Классы Fighter и Wizard; раса — Человек (без черт происхождения).
  - Опции создания и пошагового повышения уровня с выбором:
    - Воин: боевой стиль, 2 мастерства оружия, набор снаряжения; ур.2 — Action Surge (авто); ур.3 — архетип (Чемпион = крит на 19–20 / Мастер боя).
    - Волшебник: 3 заговора + 4 подготовленных заклинания; ур.2 — ячейка/доп. заклинание; ур.3 — школа магии + ячейки 2 круга + доп. заклинание.
  - Материализация листа в боевого персонажа (`build_combat_character`).
- `backend/characters_repo.py` — хранилище листов (`battle_characters`, PG/файлы).
- Эндпоинты `main.py`:
  - `GET/POST /characters-api`, `GET/PUT/DELETE /characters-api/{id}`
  - `GET /characters-api/meta/create-options`
  - `GET /characters-api/{id}/level-up/options`, `POST /characters-api/{id}/level-up`
  - `POST /characters-api/{id}/award-xp`
  - `POST /rooms/{id}/characters/from-sheet` — постановка листа в бой.
- В модель боевого `Character` добавлены `team`, `is_monster`, `subclass`, `crit_threshold` (используется в расчёте крита — Чемпион).

### Фронтенд
- `CharacterBuilderPage` — мастер создания (класс/портрет → point-buy → опции класса).
- `CharacterSheetPage` — лист персонажа: характеристики, опции, магия, способности, прогресс XP, кнопки «Повысить уровень» и «+ XP».
- `LevelUpPage` — мастер повышения уровня с выбором опций.
- `CharactersPage` — список персонажей с созданием/удалением.

### Приёмка
- `scripts/test_battle_stack.py` дополнен блоком проверки персонажей и прогрессии.

## Как проверить

Запустите стек (см. `docs/etap-0-fundament.md`, вариант A или B), затем:

### Автотест
```bash
cd battle
python3 scripts/test_battle_stack.py --no-frontend
# Блок "Character sheets & progression" — все PASS:
#   create options / fighter L1 / wizard / L2 / L3 champion / crit 19 в бою / persisted
```

### Вручную в UI (http://localhost:3001 или :8080)
1. «Персонажи» → «+ Создать персонажа».
2. Создайте **воина**: имя → характеристики (point-buy) → боевой стиль, 2 мастерства, снаряжение → «Создать».
3. На листе нажмите «+ XP» (300) → появится «Повысить уровень» → повысьте до 2 (Action Surge выдаётся автоматически).
4. Начислите ещё 600 XP → повысьте до 3, выбрав архетип «Чемпион».
5. Создайте **волшебника**: выберите 3 заговора и 4 заклинания 1 круга.
6. Перезапустите бэкенд — персонажи сохраняются (БД или файлы `backend/data/characters`).

### Проверка боя листом
- Лист материализуется в бой через `POST /rooms/{id}/characters/from-sheet` (используется в режиме «Подземелье», этап 5). У воина-Чемпиона `crit_threshold=19`, боевой стиль и мастерства применяются.

## Заметки
- Автоматическое начисление XP именно «после боя» подключается на этапах 3/5 (когда у монстров появится XP за победу). Сейчас XP начисляется через `award-xp` (кнопка «+ XP» и API).
- Характеристики редактируются только на 1 уровне (валидация point-buy).
