# Этап 2 — Заклинания в БД

Статус: завершён.

## Что сделано

### Бэкенд
- Модель заклинания расширена:
  - `effect_options` (варианты эффекта для выбора при касте),
  - `battle_ready` (готово/черновик для боевой валидации).
- Добавлены:
  - `backend/spells_repo.py` — хранилище `battle_spells` (PG/файлы),
  - `backend/spell_catalog.py` — каталог заклинаний из БД с сидированием из `backend/spells.py`.
- Реализовано сидирование: при первом запуске ~64 базовых заклинания переносятся в хранилище.
- Engine (`engine.cast_spell`) теперь:
  - читает заклинания из каталога БД,
  - поддерживает `effect_choice` — выбор варианта эффекта при касте.
- API CRUD:
  - `GET /spells`
  - `GET /spells/{id}`
  - `POST /spells`
  - `PUT /spells/{id}`
  - `DELETE /spells/{id}`
- Добавлена валидация battle-ready заклинаний (damage/save/heal поля).

### Фронтенд
- `SpellEditorPage`:
  - создание/редактирование полного статблока заклинания,
  - управление `battle_ready`,
  - добавление `effect_options` (варианты эффекта).
- `SpellbookPage`:
  - чтение из БД,
  - удаление заклинаний,
  - индикация `ready/draft`,
  - отображение вариантов эффекта.
- `CombatView`:
  - при выборе заклинания с `effect_options` появляется выбор варианта,
  - в `POST /actions/cast` передаётся `effect_choice`.

### Приёмка
- В `scripts/test_battle_stack.py` добавлен блок:
  - CRUD заклинаний,
  - создание заклинания с двумя вариантами эффекта,
  - каст в бою с `effect_choice`,
  - проверка корректного применения.

## Как проверить

```bash
cd battle
python3 scripts/test_battle_stack.py --no-frontend
```

Должны быть `PASS` в блоке **Spells CRUD & effect choice**.

### Вручную в UI
1. Откройте `/spellbook`.
2. Нажмите «+ Создать заклинание», заполните форму.
3. Добавьте 2 варианта эффекта (например `fire` и `cold`).
4. Сохраните, убедитесь, что заклинание появилось в списке.
5. В бою выберите это заклинание и переключите вариант эффекта перед кастом.

## Что важно
- Каталог заклинаний теперь не захардкожен для runtime: `spells.py` используется как seed-источник.
- Любые изменения заклинаний происходят через CRUD и сразу доступны в бою.
