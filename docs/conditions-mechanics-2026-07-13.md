# Состояния (conditions) — механика PHB 2024

_2026-07-13. Прод: 13 состояний как эффекты `effect_type='condition'` (`COND-*`). Механика —
scoped-модификаторы в `mechanics.effects[].result[]` (+ `mechanics.includes` для композиции).
Реестр/движок: [engine/conditions.ts](../frontend/src/engine/conditions.ts)._

## Итог: реализовано A–H (кроме I)

Первый проход закрыл представимую часть (Скорость 0, инициатива). Второй проход добавил новые
примитивы движка и закрыл A, B, C, D, E, F, H. G — нарративное. I отложено (трекер — отдельный компонент).

### Новые примитивы движка (единый канал `modifier`)

| Примитив | op / поле | Потребитель |
|---|---|---|
| Автопровал спаса (A) | `op:'auto_fail'` на `saving_throw`+filter | `CollectResult.autoFail`; `runSave` и оба спас-пути листа фиксируют провал без броска |
| Автокрит (B) | `op:'auto_crit'`, `scope:'target'`, `range:'melee'` | `projectedAgainst` → `runAttackRoll`: попадание рукопашной → крит |
| Дальность (C) | `range:'melee'|'ranged'` на target-проекции | `projectedAgainst(attackRange)`; тип атаки — `attackRangeFromEffect` (оружие: ammunition=дальнобойное) |
| Запрет хода (D) | `op:'deny'` на `action`/`bonus_action`/`reaction`/`concentration` | `deniedCapabilities`; `SheetActionsPanel` гейтит доступность + тултип; концентрация снимается |
| Композиция (F) | `ConditionRule.includes[]` | `conditionModifierPayloads` раскрывает транзитивно (страж циклов); `expandConditionSet` для предикатов |
| Источник (E) | `ActiveEffectEntry.sourceId` (id кастера) | `ExecuteContext.selfId` → стемпится в `applyCondition`; пикер цели блокирует очаровавшего |
| Скорость→правила (H) | `op:'set'/'multiply'` speed | `resolveCharacterRules.speed` сворачивает ops; лист передаёт состояния как condition-источники |

### Данные состояний (прод + миграция + offline-сид)

- **paralyzed / unconscious**: `auto_fail` СИЛ/ЛВК, `auto_crit` рукопашной, Скорость 0, `includes:[incapacitated]`.
- **stunned**: `auto_fail` СИЛ/ЛВК, `includes:[incapacitated]`.
- **incapacitated**: `deny` действие/бонусное/реакция/концентрация + помеха инициативы.
- **prone**: target-проекция `advantage(melee)` / `disadvantage(ranged)`.
- **grappled/restrained**: Скорость 0. **invisible**: преимущество инициативы. **blinded/poisoned/frightened**: adv/dis (G — уже верно).
- **charmed/deafened**: механика пуста (реляционное/нарративное — см. ниже), + блок цели через `sourceId`.

Скрипты прод-обновления: `scripts/content/update-conditions-2024*.mjs` (идемпотентны).

## Оставшиеся дыры / заметки

- **G — нарратив.** Оглохший (провал проверок на слух) и «нельзя видеть/слышать» — на усмотрение ГМ,
  механика не нужна. Ослеплённый: механика преимущества/помехи атак реализована (adv/dis), автопровал
  проверок на зрение остаётся нарративным.
- **Крит не удваивает кости** (пре-существующий баг движка, отдельно от auto_crit): auto_crit корректно
  помечает крит и запускает on_crit/райдеры, но удвоение костей урона — общая задача по криту.
- **Дальность = melee/ranged** (без грида): рукопашная ↔ «в пределах 5 фт», дальнобойная ↔ «дальше».
  Неоружейные атаки (заклинания) → дальность неизвестна, range-гейт закрыт (нет ложного автокрита).
- **E — реляционность.** Очарованный не может выбрать очаровавшего целью (пикер, с подсказкой). «Нельзя
  наносящими урон эффектами» и «преимущество очаровавшего на социалку» — глубже, пока не покрыто.
- **I — трекер инициативы.** Отдельный компонент (монстры/игроки), состояния листа туда не заведены —
  по решению владельца доработка пока не нужна.
