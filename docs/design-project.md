# Дизайн-проект dnd_cards: единый визуальный язык

Версия 1.0. Цель — привести весь сайт к одному фэнтезийному языку в духе
Baldur's Gate 3: конструктор персонажа как в BG3-референсах, лист персонажа —
минималистичный «классический лист» (референс Long Story Short) на тёмной теме,
и **единая система отображения сущностей** (действия/заклинания/эффекты/предметы/
виды/классы) во всех меню.

Источники, на которые опирается документ:
- `docs/battle-design-reference.md (перенесён из battle/ при удалении 2026-07-05)` + `frontend/public/design_preview.html` —
  готовый язык: хотбар-кнопки, инспектор-тултип, пипсы стоимости, состояния.
- Текущая тёмно-золотая тема Forge (`frontend/src/pages/CharacterForge.css`) и
  BG3-карточка `sp-tip` (`frontend/src/components/spellCardStyle.ts`).
- Референсы: скрины создания персонажа BG3 (рейл прогресса, сетки иконок,
  «Особенности» с круглыми иконками) и классический заполняемый лист (LSS).

---

## 1. Принципы (наследуют battle/DESIGN.md)

1. **Иконка = сущность.** Всё интерактивное — квадратная иконка; текст — подпись.
2. **Состояние читается мгновенно**: доступно / выбрано / недоступно — видом иконки.
3. **Любая иконка раскрывается при наведении** в полную карточку-инспектор.
4. **Фэнтези сдержанно**: тёмный фон, золото, свечение только как feedback.
5. **Расширяемость данными**: новый контент = иконка + запись в БД, без правок вёрстки.
6. **Один компонент — везде**: сущность выглядит одинаково в конструкторе, листе,
   библиотеке, магазине и бою.

## 2. Токены

Единый `:root` (вынести в `frontend/src/styles/tokens.css`, forge-переменные
становятся алиасами):

| Токен | Значение | Использование |
|---|---|---|
| `--bg` | `#141210` | фон страницы |
| `--panel` / `--panel-2` | `#1c1813` / `#241f18` | панели / вложенные |
| `--border` | `#6b5836` | обычные рамки |
| `--gold` / `--gold-dim` | `#d8b978` / `#c9a45f` | акцент, заголовки, выделение |
| `--text` / `--text-dim` | `#e8e0d0` / `#a99f8b` | текст / вторичный |
| `--danger` | `#d98a6a` | ошибки, «не готово» |
| `--glow` | `rgba(201,162,39,.35)` | свечение hover/selected |
| `--dt-*` | из battle | цвета типов урона |
| Радиусы | кнопка 10px, карточка 12px, панель 16px | |
| Шрифты | заголовки `Georgia, serif` (золото), текст системный | |

**Орнаментная рамка** (ключевой BG3-приём с референсов): панель с тонкой золотой
рамкой, скруглённые углы, декоративные уголки-завитки и «шпиль» по центру верхней
кромки. Реализация — компонент `OrnateFrame` (CSS-бордеры + 2–3 PNG/SVG-декора:
уголок, центральный шпиль-эмблема). Используется для: центральной панели
конструктора, панелей листа, карточки-résumé персонажа.

## 3. Единая система отображения сущностей (ядро проекта)

Три компонента в `frontend/src/components/entity/`, работают для **всех** типов:
действие, заклинание, эффект, предмет, вид, класс, черта, предыстория.

### 3.1 `<EntityIcon>` — квадратная иконка (формат «как у заклинаний»)

По образцу `.ab` из battle (56×56) и `forge-spell-icon`:
- Размеры: `sm` 32, `md` 44, `lg` 56, `xl` 72 (сетки выбора в конструкторе).
- Скруглённый квадрат 10px, «каменная» подложка (radial-gradient), рамка `--border`.
- Состояния: `ready` (золотой ободок, hover: подъём + свечение), `selected`
  (золотое кольцо 2px + glow), `disabled` (grayscale, not-allowed), нейтральное.
- **Бейдж** (нижний-правый): заряды/уровень слота/количество (для предметов).
- **Уголок** (верхний-левый): цвет типа ресурса (бонусное — оранжевый, реакция —
  синий) — как в battle §3.
- Иконка с `onError`-фолбэком на `/default_image.png` — никогда не «дырка».
- Опциональная подпись под иконкой (`label` — как сетки BG3: «Варвар», «Бард»…).

### 3.2 `<EntityInline>` — иконка + текст (формат «как у эффектов»)

По образцу `ForgeAbilityLine`, обобщённый:
- Круглая или квадратная мини-иконка 20–24px + название (золотое, с подчёркиванием
  как ссылка) + опциональный суффикс-источник («· вид Дварф»).
- Используется в списках: способности на листе, «даёт» у предыстории, сводка Forge.

### 3.3 `<EntityHoverCard>` — единый инспектор при наведении

Оба формата (3.1 и 3.2) при hover показывают **одну и ту же** полную карточку.
База — семейство `sp-tip` (`spellCardStyle.ts`), структура battle §4:
большая выезжающая иконка → заголовок + подтип → стат-строки (Атака/Спасбросок/
Урон/Лечение — каждая на своей строке, цвета типов урона) → описание (`**жирный**`
золотом) → мета (🎯 ⏱ ◈ ⊙) → плашка стоимости с иконками ресурсов.

Маппинг сущностей на карточку:
| Сущность | Карточка |
|---|---|
| Заклинание | готово — `SpellPreview` |
| Действие | готово — `ActionPreview` |
| Эффект | **переделать** `EffectPreview` со slate-синего на `sp-tip` |
| Предмет | новый `ItemHoverCard` на sp-tip: слот/вес/цена в мета, кость урона / КБ в стат-строках |
| Вид/класс/черта/предыстория | существующие Preview, постепенно привести рамку/фон к токенам |

Позиционирование: `position:fixed`, кламп по краям окна, над/под курсором
(логика уже есть в `ForgeAbilityLine`/листе — вынести в хук `useHoverCard`).

**Депрекация**: `forge-effect-card` (синий поповер) и slate-стиль `EffectPreview`
заменяются на sp-tip; `EffectIcons`/`ActionIcons` — на `EntityIcon`.

## 4. Экран «Создание персонажа» (перестройка Forge под BG3)

Референс — скрины BG3. Раскладка:

```
┌──────────────────────────────────────────────────────────────┐
│ [Рейл прогресса]   [Центральная панель OrnateFrame]  [Résumé]│
│  ◉ Биография        Заголовок секции («Класс»)        карточка│
│  ◉ Вид · Дварф      Сетка выбора: EntityIcon xl +     персонажа│
│  ◉ Подвид · Щитовой   подписи, выбранный — золотая     (свернув.│
│  ◉ Класс · Монах      рамка                            иконка) │
│  ◉ Происхождение    Название выбранного + описание            │
│  ◉ Характеристики   [Действия: EntityIcon-строка]             │
│                     [Особенности: EntityIcon-строка]          │
└──────────────────────────────────────────────────────────────┘
```

1. **Рейл прогресса** (замена `forge-nav`): вертикаль слева, каждый шаг —
   круглый медальон-«печать» (штамп с галочкой как в BG3) + две строки:
   название шага и текущий выбор золотом («Раса / Дварф»). Состояния:
   пройден (золотая печать), активен (подсвечен), не начат (тусклый).
   Порядок шагов: Общее → Вид → Подвид (если есть) → Класс → Предыстория →
   Черта → Характеристики → Заклинания (для кастеров) → Владения (сводка).
2. **Центральная панель** — `OrnateFrame` с шпилем-эмблемой активного шага
   сверху. Внутри: заголовок (Georgia, по центру), сетка `EntityIcon xl`
   с подписями (4 в ряд), под ней название выбранного + его описание
   (текст-центр, как BG3), затем полосы **«Действия»** и **«Особенности
   класса/расы/подрасы»** — рамка-лента с рядом `EntityIcon md` (hover —
   полная карточка). Это заменяет текстовые списки эффектов.
3. **Подвид** — отдельный шаг рейла (появляется после выбора вида): та же
   сетка из lineage-вариантов + «Особенности подрасы».
4. **Характеристики** — экран «Очки умений» BG3: строки СИЛ…ХАР со
   значением и ± (point-buy 27 очков — включить существующую логику
   `getPointBuyCost`), справа две колонки чекбоксов **«Назначить бонус +2/+1»**
   (бонусы предыстории/вида по правилам 2024 — см. план §5), кнопки
   «Очистить» и «Оптимальные значения» (пресет по primary_abilities класса),
   снизу панель «Навыки с умением» со ссылкой «Изменить».
5. **Résumé-карточка** (правый верх, сворачиваемая) — как 4-й скрин:
   эмблема класса, «Щитовой дварф / Уровень 1 Монах», строка характеристик
   СИЛ…ХАР, инициатива и HP в медальонах, полоса «Действия» (EntityIcon),
   блок «Умения»: Оружие/Броня/Навыки списком с золотыми значениями.
   Заменяет текущую левую сводку `forge-summary`.

## 5. Лист персонажа (референс LSS, тёмная тема)

Минималистичный «бумажный» лист, переведённый в тёмно-золотые токены.
Каждый блок — тонкая рамка `--border`, скруглённые углы, узкий заголовок-капс.

Сетка (desktop, 3 колонки):

```
┌ Шапка: Имя │ Происхождение │ Класс+уровень │ Вид │ [Уровень-медальон] ┐
├ Ряд статов: [БОНУС УМЕНИЯ +2] [ИНИЦИАТИВА +3] [СКОРОСТЬ 30]           │
│             [П.ВНИМАНИЕ 14] [СОСТОЯНИЯ …]  [КБ-щит 16] [HP 11/11 +врем]│
├─ Колонка 1 ──────┬─ Колонка 2 ────────────┬─ Колонка 3 ───────────────┤
│ 6 блоков хар-к:  │ ОРУЖИЕ И ФОКУСЫ        │ СПОСОБНОСТИ КЛАССА        │
│ [мод крупно]     │ таблица: EntityInline  │ EntityInline-список       │
│ [значение]       │ + бонус + урон/вид     │ (иконка, название, свёрн. │
│ ○ спасбросок     │ (из экипировки)        │ описание; hover — карточка)│
│ ○ навыки со      │ ИНВЕНТАРЬ/ПРЕДМЕТЫ     │ ВИДОВЫЕ ОСОБЕННОСТИ       │
│   значениями     │ EntityInline + вес,    │ EntityInline-список       │
│ ИСТОЩЕНИЕ        │ слоты надетого         │ ЧЕРТЫ                     │
│ ГЕРОИЧ. ВДОХН.   │                        │ ЗАКЛИНАНИЯ (сетка         │
│ УМЕНИЯ И ЯЗЫКИ   │                        │ EntityIcon по уровням)    │
└──────────────────┴────────────────────────┴─ ЖУРНАЛ (лог бросков) ────┘
```

Требования:
- **Блок характеристики** как в LSS: крупный модификатор в круге, значение,
  под ним спасбросок и навыки этой характеристики с бонусами; владение —
  закрашенная точка. Тултип каждого числа — `ValueBreakdown` (движок F2).
- **КБ** — щит-силуэт с числом (breakdown по hover: доспех+щит+эффекты).
- **HP-блок**: текущие/макс, временные, кость хитов (d10), испытания против
  смерти (3+3 точки) — визуально как в референсе.
- **Поля способностей** — фиксированные секции «Способности класса»,
  «Видовые особенности», «Черты», «Предметы»: внутри только
  `EntityInline`-строки (никаких карточек-простыней); полная информация — hover.
- Панель runtime (ресурсы-чипы, «Новый ход», отдыхи) — над журналом,
  чипы ресурсов — `EntityIcon sm` с бейджем текущее/макс.

## 6. Остальные экраны

- **Библиотека карт**: сетка `EntityIcon xl` с подписями + переключатель
  «иконки/карточки» (карточки — текущие Preview). Фильтры в тонкой золотой панели.
- **Магазин**: товар = `EntityInline` + цена; hover — `ItemHoverCard`.
- **Инвентарь листа**: `EntityInline` + вес/кол-во; слоты экипировки — ряд
  `EntityIcon md` с силуэтами-плейсхолдерами пустых слотов (голова/тело/руки).
- **Конструкторы сущностей** (creator-страницы) остаются светлыми рабочими
  инструментами — их не переводим на тёмную тему в этом проекте (кроме
  live-превью, которые уже в sp-tip).

## 7. Иконки и ассеты

- Пути: `/icons/actions/...`, `/icons/spells/...`, `/icons/resources/...`,
  предметы/сущности — `image_url` из БД. Фолбэк всегда `/default_image.png`.
- Медальоны рейла, уголки OrnateFrame, щит КБ, шпили — набор из ~6 SVG/PNG
  в `/icons/ui/` (сгенерировать через ImageStudio в стиле референсов:
  золотая гравюра на прозрачном фоне).
- Круглые иконки «особенностей» BG3 (синие глифы) — стиль для эффектов:
  тёмно-синий круг + белый глиф; уже совпадает с текущими иконками эффектов.

## 8. План внедрения (шаги для легковесной модели)

Порядок минимизирует риск: сначала система сущностей, потом экраны.
Правила исполнителю — как в §5 mvp-transition-plan (build vite, тесты зелёные,
проверка через `vite preview` + браузер).

- **DS1. Токены и OrnateFrame.** `styles/tokens.css` (§2), forge-переменные —
  алиасы; компонент `OrnateFrame` (+ ассеты-уголки из §7). КП: Forge и лист
  рендерятся без визуальных регрессий (скриншот-сравнение вручную).
- **DS2. `entity/` компоненты.** `EntityIcon` (все размеры/состояния/бейдж/
  уголок), `EntityInline`, `useHoverCard`, `EntityHoverCard`-роутер по типу
  сущности. КП: сторибук-страница `/design-preview` (роут dev-only) со всеми
  состояниями; hover работает с обоих форматов.
- **DS3. Эффекты и предметы на sp-tip.** `EffectPreview` → sp-tip;
  `ItemHoverCard` (слот/вес/цена/кость/КБ). Замена `forge-effect-card`-поповера
  на `EntityHoverCard`. КП: наведение на эффект в листе и Forge показывает
  sp-tip; предмет в магазине — ItemHoverCard.
- **DS4. Лист персонажа — LSS-раскладка.** Перевёрстка `CharacterSheetMVP`
  по §5: блоки характеристик, щит КБ, HP-блок, поля способностей на
  `EntityInline`, заклинания сеткой `EntityIcon`. Runtime-панель и журнал —
  на свои места. КП: скрин соответствует схеме §5; все числа с breakdown.
- **DS5. Forge: рейл + центральная панель.** Медальонный рейл, OrnateFrame,
  сетка `EntityIcon xl` для вида/класса/предыстории/черты, полосы
  «Действия»/«Особенности». Подвид — отдельный шаг. КП: скрин-сравнение с
  BG3-референсом; все выборы работают как раньше (создание проходит).
- **DS6. Forge: экран характеристик BG3.** Point-buy 27 + колонки «Назначить
  бонус +2/+1» (источник бонусов — предыстория 2024; связать с
  `background.ability_scores`), «Оптимальные значения», панель навыков.
  КП: создание с point-buy и бонусами сохраняет корректные значения.
- **DS7. Résumé-карточка.** Правая сворачиваемая карточка персонажа (§4.5)
  вместо текстовой сводки. КП: скрин, данные живые.
- **DS8. Библиотека/магазин/инвентарь.** Унификация по §6. КП: сквозной
  сценарий покупка→инвентарь→экипировка в новом виде.

Зависимости: DS1 → DS2 → DS3 → (DS4, DS5 параллельно) → DS6 → DS7 → DS8.
DS4–DS8 не блокируют движковые фазы E/F — вести параллельно, но F1 (панель
действий листа) делать уже на компонентах DS2.

## 9. Критерий готовности дизайн-проекта

- Ни одного места, где действие/заклинание/эффект/предмет отображаются
  «по-своему»: только EntityIcon / EntityInline / EntityHoverCard / Preview.
- Конструктор персонажа визуально соответствует BG3-референсам (рейл,
  сетки, особенности, résumé).
- Лист персонажа соответствует LSS-схеме §5, все значения с разбивкой.
- battle/ и основной сайт используют один набор токенов и одну структуру
  инспектора.
