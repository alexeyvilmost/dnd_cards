# Контент-свип narrative→choice — отчёт (2026-07-09)

Перевод эффектов, где выбор описан ТЕКСТОМ («по вашему выбору»), в настоящие исполнимые
`choice`. Задача — не «переписать всё скопом», а конвертировать то, что движок **реально
применяет**, с тестами, и честно перечислить остальное с причиной.

## Итог
- **Конвертировано и протестировано в этом заходе: 3 эффекта** (+ 2 из слайсов 4/5 как регрессия).
- Механика выборов теперь покрывает домены: `skill, ability, tool, instrument, artisan_tool,
  damage_type, feat, spell, sense/speed, explicit` и 3 контекста (создание / левелап / in-play).
- Гейты: **tsc 0, 292 vitest** (в т.ч. 15 тестов свипа), **83 mvp**. Схема-валидация — VALID.

## Граница: почему не всё сразу
Резолвер сборки применяет только **build-гранты**: `grant_proficiency / grant_expertise /
grant_language / grant_feat / grant_spell / grant_ability_score / grant_sense / grant_speed /
grant_effect`. **Runtime-пейлоады** из выбора (`resistance / modifier / condition / temp_hp /
damage / resource`) НЕ применяются: `collectPassiveMechanics` отдаёт механику эффекта как есть,
не разворачивая `choice`. Поэтому:
- эффекты с **build-грантом** — конвертируются и проверяются **сейчас**;
- эффекты с **runtime-грантом** ждут движка (развернуть выбранный runtime-пейлоад в пассив),
  либо моделируются через `grant_effect` + под-эффект на каждый вариант (отдельная работа);
- эффекты с **выбором по действию/триггеру** (бонусным действием / при Ярости / реакцией) ждут
  подслайса «in-play by-trigger» (выбор в момент исполнения действия).

## Отложено (с причиной)
Категоризация ориентировочная — точный блокер уточняется по месту. Всего в аудите 42 эффекта;
за вычетом сделанных остаётся ~37, они распадаются на 3 группы.

### A. Runtime-грант — нужен движок (или `grant_effect`+под-эффекты)
Выбор даёт сопротивление / временные хиты / модификатор / состояние — их пассивный сборщик
не разворачивает из `choice`.
- EFFECT-0235 Родство со стихией (сопротивление типу урона + бонус к урону заклинания)
- EFFECT-0139 Устойчивость Исчадия (сопротивление типу урона, меняется после отдыха)
- EFFECT-0160 Воплощение стихий (сопротивление)
- EFFECT-0092 Природный оберег (иммунитет/сопротивление)
- EFFECT-0144 Небесное сопротивление (временные хиты)
- EFFECT-0158 Стихийный выброс, EFFECT-0231 Откровение во плоти
> Путь: либо (1) научить `collectPassiveMechanics`/рантайм разворачивать выбранный пейлоад,
> либо (2) на каждый вариант завести под-эффект-пассив и выбирать через `choice(source:effect)`
> → `grant_effect` (это УЖЕ разворачивается и попадает в пассивы, проверить применение).

### B. Выбор по действию/триггеру — нужен подслайс «in-play by-trigger»
Выбор моментальный, в момент исполнения действия/триггера, а не «стоячий».
- при Ярости: EFFECT-0028 Ярость дикой природы, EFFECT-0031 Сила дикой природы, EFFECT-0035 Путешествие по Древу
- бонусным действием: EFFECT-0082 Третий глаз, EFFECT-0056 Оплот силы, EFFECT-0039 Фанатичное присутствие,
  EFFECT-0228 Телепатическая речь, EFFECT-0085 Звёздный облик, EFFECT-0026 Пугающее присутствие,
  EFFECT-0016 Мантия вдохновения, EFFECT-0212 Исключительная дрессировка, EFFECT-0100 Гнев моря
- реакцией / по триггеру: EFFECT-0245 Подчинение удачи, EFFECT-0145 Жгучее возмездие,
  EFFECT-0020 Вдохновляющее перемещение, EFFECT-0152 Техника открытой ладони
- действием: EFFECT-0090 Помощь земли, EFFECT-0120 Сияние рассвета, EFFECT-0111 Сохранение жизни,
  EFFECT-0170 Гнев природы, EFFECT-0242 Заводная кавалькада
- после каста: EFFECT-0247 Укрощённая волна, EFFECT-0180 Воодушевляющая кара, EFFECT-0066 Построение заклинаний
- прочее: EFFECT-0149 Прикосновение целителя, EFFECT-0118 Улучшенная двуличность,
  EFFECT-0217 Потустороннее очарование, EFFECT-0129 Пробуждённый разум, EFFECT-0126 Туманное исчезновение,
  EFFECT-0191 Псионическая сила (кости по уровню — не выбор)

### C. Сложный контент (моделируется сейчас, но объёмно)
- EFFECT-0089 Заклинания Круга земли — `choice(context:in_play)` из 4 типов земли, каждый → grant_spell
  по уровням (нужно сопоставить ~16 слагов заклинаний). Механизм готов, дело в объёме контента.
- EFF-wizard-scholar Учёный — «экспертиза ИЛИ инструмент каллиграфа»: выбор ИЗ РАЗНЫХ типов гранта
  (`explicit` с per-item grants: часть вариантов grant_expertise, один grant_proficiency tool).

## Как проверить (приёмка)
```
cd frontend
npx vitest run src/mechanics/contentSweep.test.ts    # 15/15 (5 эффектов × 3 проверки)
npx vitest run                                        # 292 зелёных (полный набор)
npx vitest run --config vitest.mvp.config.ts          # 83 (валидирует прод-контент)
```
Прод уже обновлён (`node scripts/content/seed-content-sweep.mjs --apply`). Живой прогон:
персонаж с чертой «Ремесленник»/«Музыкант» → выбор инструментов; Бард с «Магическими
открытиями» → выбор 2 заклинаний; Воин с доп. боевым стилем → выбор стиля.

## Дальше
Разблокировать группу A (runtime-choice-application в движке) — самый большой рычаг: сразу
оживит сопротивления/бонусы из выборов. Затем группа B (подслайс by-trigger). Оба — после
готовности движка, как и договаривались по слайсу 6.
