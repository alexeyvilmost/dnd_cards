## Правила исполнения (читать до первой правки)

1. **§3 (парадигмы) и §9 (грабли) KB — обязательное чтение.** Нарушение парадигмы = задача не принята, даже если DoD зелёный.
2. **Не включено в план сознательно:** авторизация (парадигма №7, KB-198); удаление визуально отображаемого кода (парадигма №6); массовый автолинковщик по именам заклинаний (KB-129); кости хитов (§8.4.1).
3. **Не «чинить» §8.5 (19 опровергнутых гипотез)** — там мины обратного знака.
4. **Задачи `kind: data`** — обязательный протокол: скрипт в `scripts/content/<имя>.mjs` через `scripts/content/api.mjs` → применить к проду → `node scripts/content/export-prod.mjs` → закоммитить и скрипт, и обновлённый снапшот. Скрипт **идемпотентен** (UPDATE-ветка, не create-only — грабли KB-088).
5. **Гейт сборки** — `cd frontend && npx vite build` (канонический), `./node_modules/.bin/tsc --noEmit`, `cd backend && go build ./...`. `npm run lint` не существует (KB-206) — не пытаться.
